# Alicia 机器人智能抓取仿真系统 — 项目需求文档

> **环境**: WSL2 Ubuntu 22.04 + ROS 2 Humble + Gazebo Fortress (Ignition)
> **工作目录**: `~/RobotSim/`（从零构建）
> **版本**: v2.1 — 纯 Alicia 6-DOF 重构版

---

## 一、项目概述

### 1.1 项目目标

为 **Alicia-D v5.6 六轴机械臂（含原始平行夹爪）** 构建一套完整的 **感知-决策-执行（PDE）闭环仿真系统**，使机器人能在 Gazebo 仿真环境中：

1. 通过俯视 RGB-D 相机识别料仓内的彩色物体
2. 由状态机自动决策抓取顺序和策略
3. 使用 MoveIt2 规划运动路径并通过 `gz_ros2_control` 驱动 Gazebo 中的物理关节
4. 完成"识别 → 移动到预抓取位 → 夹爪闭合 → 搬运到放置位 → 夹爪松开"的完整闭环

### 1.2 系统范围

- **仅限仿真**：所有交互在 Gazebo Sim 中完成，不涉及真实硬件
- **机器人构型**：Alicia-D v5.6 六轴机械臂 + 原始 100mm 平行夹爪（`Gripper` 棱柱关节 + `left_gripper`/`right_gripper`），**不使用 SO-ARM 夹爪**
- **运动学链**：`base_link → Link1~Link6 → left_gripper/right_gripper → gripper_center`
- **活动关节**：Joint1~Joint6（旋转）+ Gripper（棱柱，夹爪开合）+ right_finger（mimic Gripper）
- **目标物体**：3 个简单几何体（红色圆柱、黄色方块、棕色球体）
- **场景布局**：
  - 机器人固定在世界原点偏右 (0.20, 0, 0)
  - Bin A（取料仓）在机器人前方左侧
  - Bin B（放料仓）在机器人前方右侧
  - 俯视相机固定在 Bin A 正上方

### 2.2 ROS 2 包结构

| 包名 | 类型 | 职责 |
|------|------|------|
| `alicia_d_descriptions` | ament_cmake | 机器人 URDF + Mesh 描述文件（原始包，含夹爪 mesh）|

## 三、功能需求

### 3.1 物理仿真层（Phase 1）

#### FR-1.1 机器人底座稳定性
- **描述**：机器人底座必须刚性固定在世界坐标系中
- **实现要求**：在世界 SDF 文件中，使用 `<joint type="fixed">` 将 `alicia_d::base_link` 锚定到 `world`
- **验收标准**：启动仿真后 30 秒内，`base_link` 的位姿变化量 < 0.001m

#### FR-1.2 关节阻尼
- **描述**：所有活动关节（Joint1~Joint6、Gripper）必须有合理的阻尼参数
- **验收标准**：仿真启动后无控制器的情况下，关节角速度不超过 0.5 rad/s

#### FR-1.3 动态物体
- **描述**：料仓内的 3 个测试物体必须是动态的（`<static>false</static>`），具有合理的质量和惯性
- **验收标准**：物体在料仓中静止不穿模，可以被施加外力后移动

#### FR-1.4 Gazebo model.sdf
- **描述**：需要为纯 Alicia-D（含原始夹爪）创建新的 Gazebo model.sdf
- **实现要求**：
  - 基于 `Alicia_D_v5_6_gripper_100mm.urdf` 转换为 SDF 格式
  - 包含 base_link、Link1~Link6、left_gripper、right_gripper、gripper_center
  - 包含 Joint1~Joint6（revolute）、Gripper（prismatic）、right_finger（prismatic, mimic）
  - 移除 SO-ARM 相关的所有 link/joint/frame/collision
  - 包含 `gz_ros2_control` 插件配置
  - 包含 `JointStatePublisher` 插件
  - **不使用** `<gravity>false`、`<kinematic>true`、`<velocity_decay>` 等 workaround

### 3.2 控制层（Phase 2）

#### FR-2.1 gz_ros2_control 统一硬件接口
- **描述**：用 `gz_ros2_control` 替代 `fake_components/GenericSystem`
- **实现要求**：
  - 创建新的 `alicia_d.ros2_control.xacro`，硬件插件使用 `gz_ros2_control/GazeboSimSystem`
  - 关节列表：Joint1~Joint6 + Gripper
  - 命令接口：`position`
  - 状态接口：`position`、`velocity`
- **验收标准**：`ros2 topic echo /joint_states` 能看到 7 个关节的真实状态

#### FR-2.2 统一控制器配置
- **实现要求**：
  - `Alicia_controller`：`joint_trajectory_controller`，关节 Joint1~Joint6
  - `Gripper_controller`：`joint_trajectory_controller`，关节 Gripper
  - `joint_state_broadcaster`
- **验收标准**：`ros2 control list_controllers` 显示三个控制器均为 `active`

#### FR-2.3 统一 Launch 文件
- **启动序列**：
  1. Gazebo 仿真（加载 bin_scene.sdf）
  2. robot_state_publisher（发布 URDF）
  3. gz_ros2_control → controller_manager
  4. spawn joint_state_broadcaster → Alicia_controller → Gripper_controller
  5. MoveIt move_group
  6. ros_gz_bridge（相机数据桥接）
  7. static_transform_publisher（相机 TF）
  8. RViz（可选）
- **验收标准**：`ros2 launch alicia_moveit_config sim_demo.launch.py` 一键启动

### 3.3 感知层（Phase 3）

#### FR-3.1 RGB-D 图像桥接
- 桥接 `/rgbd_camera/image`、`/rgbd_camera/depth_image`、`/rgbd_camera/camera_info`
- 验收：`ros2 topic hz /rgbd_camera/image` ≈ 5 Hz

#### FR-3.2 相机 TF 链
- `static_transform_publisher` 发布 `world → rgbd_camera_frame`
- 验收：`ros2 run tf2_ros tf2_echo world rgbd_camera_frame` 成功

#### FR-3.3 HSV 色彩感知
- 保留现有 `color_perception_node.py` 的 HSV 分割逻辑
- 发布 Topics：`/perception/current_target_visible`、`/perception/current_target_point_world`、`/perception/current_target_pregrasp_world`、`/perception/current_target_name_echo`

### 3.4 决策层

#### FR-4.1 有限状态机
- 保留 18 态状态机 `task_state_machine_node.py`
- 修改 `VIRTUAL_GRASP` / `VIRTUAL_RELEASE` 状态为实际发送夹爪开合指令

### 3.5 执行层

#### FR-5.1 修复执行节点接口（重写 `move_to_pregrasp_node.py`）
- **订阅**：
  - `/task/enable_pregrasp` (Bool)、`/perception/current_target_pregrasp_world` (PointStamped)
  - `/task/goal_pose_world` (PoseStamped)、`/task/execute_goal_pose` (Bool)
- **发布**：`pregrasp_running/done/success` + `goal_pose_running/done/success`（各 Bool）
- **关键**：PointStamped → PoseStamped 转换（补全抓取朝向 roll=180°）

#### FR-5.2 夹爪控制
- 通过 `Gripper_controller` 发送夹爪开合指令
- 闭合：`Gripper → 0.05`（最大行程），松开：`Gripper → 0.0`

## 六、Topic 接口定义

### 6.1 感知层输出

| Topic | 类型 | 频率 | 描述 |
|-------|------|------|------|
| `/perception/current_target_visible` | Bool | 2 Hz | 目标是否可见 |
| `/perception/current_target_point_world` | PointStamped | 2 Hz | 目标世界坐标 |
| `/perception/current_target_pregrasp_world` | PointStamped | 2 Hz | 预抓取位置（上方 0.10m）|
| `/perception/current_target_name_echo` | String | 2 Hz | 当前跟踪目标名 |

### 6.2 决策层输出

| Topic | 类型 | 描述 |
|-------|------|------|
| `/task/current_target_name` | String | 指示感知关注目标 |
| `/task/enable_pregrasp` | Bool | 触发 pregrasp 运动 |
| `/task/goal_pose_world` | PoseStamped | 放置目标位姿 |
| `/task/execute_goal_pose` | Bool | 触发放置运动 |
| `/task/gripper_command` | Float64 | 夹爪开合指令（0.0=开, 0.05=闭）|

### 6.3 执行层反馈

| Topic | 类型 | 描述 |
|-------|------|------|
| `/task/pregrasp_running/done/success` | Bool | pregrasp 运动状态 |
| `/task/goal_pose_running/done/success` | Bool | 放置运动状态 |

## 七、验收标准

| 阶段 | 验收项 | 通过条件 |
|------|--------|----------|
| Phase 1 | 底座稳定 | 仿真 30s 后 base_link 无漂移 |
| Phase 1 | 物体动态 | 物体可被推动且不穿模 |
| Phase 2 | 控制器就绪 | 3 个控制器均 active |
| Phase 2 | 关节可控 | RViz 拖动关节 → Gazebo 同步 |
| Phase 2 | 夹爪可控 | 发送夹爪指令 → Gazebo 中夹爪开合 |
| Phase 3 | 感知识别 | RViz Marker 正确标注目标 |
| 全链路 | PDE 闭环 | 状态机驱动完整抓取-放置流程 |
